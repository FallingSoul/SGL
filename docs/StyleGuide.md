# 代码规范文档

## 代码格式
+ 缩进: 默认四空
+ 换行: 非必要不可换行,必要是指利于阅读
+ 括号: 不可以省略,保证运算顺序歧义
+ 特别: 采用`#ifndef...#define...#endif//!...`代替`#program once`
	+ 其中的`...`样式为`__SGL_***__`,`***`表示当前结构位置
	+ e.g. `#ifndef __SGL_CORE_WINDOWS__#define SGL_CORE_WINDOWS__#endif//!__SGL_CORE_WINDOWS__`
## 命名约定
+ 语言: 英文
+ 变量: 字母采用小写,多单词之间可以用'_'来分割
	+ 全局变量: 加上前缀'sgl'
	+ 局部变量: 没有前缀
	+ 私有变量: 加上前缀'_'
	+ 公共变量: 没有前缀
	+ 其他变量: 根据情况适当命名
+ 函数: 字母采用小写,多单词之间可以用'_'来分割
	+ 一般函数: 加上前缀'sgl'
	+ 私有函数: 加上前缀'_'
	+ 公共函数: 没有前缀
	+ 其他函数：根据情况适当命名
+ 类: 统一加上前缀'sgl',其余采用大驼峰命名
+ 结构体: 统一加上前缀'sgl',其余采用大驼峰命名
+ 枚举: 统一加上前缀'sgl',其余采用大驼峰命名
## 注释规则
+ 语言: 没有限制,但是最好英文
+ 内容: 准确无误,契合代码,不冗余
## 代码结构
+ 采用命名空间来构造结构,所有命名空间放在`sgl`下
+ 基础性代码放在`sgl::core`下
+ 框架代码放在`sgl`下
+ 结构与结构之间不应过分依赖
## 代码责任
+ 自己负责自己的代码,遵守规范
+ 在自己的代码中发现bug,如果没有和别人的代码联系,就自行解决;如果和别人的代码有联系,请先联系他共同处理
+ 在别人的代码中发现bug,必须告诉他具体情况,可以帮助修正,但需要征得同意
+ 接手别人的bug或者代码后,责任就转交了,也可以共同承担责任
+ 对于较为棘手的bug,要先做记录,提交issue,不可以擅自修正
+ 对于提交的issue属于团队责任,需要共同处理,也可以由个人自愿承担责任
